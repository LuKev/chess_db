25. If uploads/seed queue but no games appear, the most common cause is the worker not running or BullMQ/Redis connection miswiring (imports stay `queued`).

The app-shell left sidebar can also be collapsed from the header and its state persists in browser storage.
28. Bulk collection/tag actions on `/games` are implemented, but they only enable when at least one game is selected and at least one collection/tag exists. Keep `collections`/`tags` queries invalidated alongside `games` after bulk assignment changes or the `/games` counts/options will look stale even when the write succeeded.
29. Password reset UX on `/login` is a two-step flow: request token by email, then submit token + new password (step 2 disabled until a request has been made or a token is present).
30. Production outage pattern (observed 2026-02-18): `https://api.kezilu.com` returns Railway edge `502` and web login shows `TypeError: Failed to fetch` when Railway `Postgres` is crashed due disk exhaustion (`pg_wal/xlogtemp.*: No space left on device`); API then crashes with DB `ETIMEDOUT`.
31. Storage-growth hotspot: `/Users/kevin/projects/chess_db/apps/api/migrations/0005_chessbase_features.sql` creates `game_positions` with several indexes (`user+fen`, `user+material`, `user+game+ply`, `user+next_fen`); this is roughly one row per ply per imported game and can dominate small Postgres volumes quickly.
32. Product direction is now explicit: optimize for a serious single-user prep workflow first, not collaboration/training. Position/opening indexing is background-by-default; imports should finish after canonical game writes and then queue backfills instead of writing `game_positions`/`opening_stats` synchronously.

34. Web board rendering now uses a shared `ChessPiece` image component (`apps/web/components/ChessPiece.tsx`) backed by vendored open-source piece assets in `apps/web/public/pieces/` from `chessboard-element@1.2.0` (MIT). Piece asset URLs must be prefixed through `apps/web/lib/basePath.ts` (`addBasePath`) so production `/chess_db/...` deploys do not request broken root-relative `/pieces/...` links.
35. Playwright replay assertions must not use square `textContent` anymore because boards render piece images; compare each square's image `src` (or another piece-presence attribute) instead. On the public `/login` page, password-reset E2E steps should wait for logout to settle into a stable signed-out page before filling controlled inputs, or the value can be wiped by a late rerender and make the click look like a no-op.
36. `/diagnostics` game viewer had a real async race: `openGameViewer()` rendered the viewer before annotations finished loading, and the later annotation hydrate could reapply the default line/cursor and snap the board back to ply 0. Keep late annotation restores guarded so they only apply if the same game is still open and the user has not moved away from the initial cursor/line.
